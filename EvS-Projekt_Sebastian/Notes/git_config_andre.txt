Falls mehrere Leute (A, B) die selbe Datei bearbeiten, ist pushen nur mehr
für einen möglich, den schnelleren (zb. A). 

A pusht die Datei ins Remote Repo, versucht nun B zu pushen bekommt er 
einen "Konflikt error". 

B muss jetz
zuerst vom Remote Repo pullen, die gepullte Datei (Datei von A) mit seiner 
vergleichen (im schlechtesten Fall Line by Line), ergänzen und dann die
gemergte Datei ins Repo pushen. 

In meinem Fall wurde die selbe Datei die von Adrian und mir geändert wurde
beim pullen (von mir) automatisch mit meiner Datei (lokal) gemerged, 
allerdings mit zusätzlichen git Informationen in der Datei (Zeile die sich 
unterscheidet, Hashwerte usw...).
Als ich die Datei bereinigt, Adrians und meine Zeile geprüft habe, war
pushen wieder möglich.
